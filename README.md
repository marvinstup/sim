문서 1. 모델 비교 보고서

같은 입력을 네이토 모델 드롭다운에서 바꿔가며 비교 (예: GPT-5 / Claude Opus 4.8 / Gemini 3.1 Pro)
네이토에서 기록

순서: 모델 A / 모델 B / 모델 C
1. 정확성(사실·수치): 4 / 5 / 3  → C는 '확인 필요' 항목 일부 누락
2. 한국어 자연스러움: 4 / 4 / 5
3. 형식 준수: 3 / 5 / 4  → B가 불릿 구조 가장 정확
4. 긴 문맥 유지: 4 / 5 / 3
5. 속도·안정성: 5 / 4 / 4
합계 → 20 / 23 / 19  →  최종 선정: 모델 B

선정이유 (회의록 요약 과업 특성과 연결)
1) 회의록 요약은 결정사항/Action Items/리스크/후속 일정 4섹션 형식이 핵심인데, B모델이 매 턴 형식을 가장 일관되게 지킴
2) 여러 번 조건이 바뀌어도 기존 결정사항을 유지해야 하는 과업 특성상, 긴 문맥 유지 점수가 높은 B모델이 적합
3) C모델은 "확인 필요" 항목 일부 누락 — 회의록 요약에서 누락은 실무 오류로 직결되므로 제외

문서 2. 시스템 설계 문서

누가: 주간 회의를 운영하는 팀 리드, PM
뭘 넣으면: 회의 메모 또는 녹취 텍스트
뭐가 나오는지: 결정사항 / Action Items / 리스크 / 후속 일정 4섹션
----------------------------------

재사용 입력 템플릿 
[업무 과업] 회의록 요약
[원문] (회의 메모/녹취 텍스트)
[원하는 결과] 결정사항 / Action Items(담당·기한) / 리스크 / 후속 일정
[톤] 간결, 사내 공유용
[금지] 추측성 표현 금지, 인물 실명 대신 역할명 사용
[확인 질문 규칙] 정보가 부족하면 최대 3개까지 질문 후 요약 시작
------------------------
페르소나 + 시스템 프롬프트
이름: 정리봇
역할: PM 출신 회의록 정리 코치
전문 분야: 결정사항/Action Item 추출, 사내 공유 문서화
말투: 간결한 존댓말, 군더더기 없음
금지: 추측을 사실처럼 단정하기, 실명 노출
우선순위: 정확성 > 친절함 (부족하면 먼저 질문)
----------------------------------
예시 3개 (1개는 되묻기)
-------------------------

 v2 개선

 - Action Items는 "담당(역할명) · 기한 · 작업" 형식
 [안전장치]
 - 정보 부족 시 임의로 채우지 말고 최대 3개까지 확인 질문 먼저
 - 모르면 "확인 필요"로 표기
[사실·수치 규칙]
 - 원문에 없는 날짜·수치·담당자를 만들지 않는다
 - 근거가 원문에 있으면 그대로, 없으면 '확인 필요'로 분류
[금지] 인물 실명 대신 역할명, 추측성 단정 금지
---------------------------------
